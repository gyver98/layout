  }
    
  /* The default behavior of Grid Auto Flow is to layout the elements by row, 
     working along the row until there are no more slots then moving on to the next row.
     This default behavior can be changed by using the grid-auto-flow property.
     If you specify column then the elements will be laid out column by column. */
  
  .grid-container {
      display: grid;
      grid-template-columns: 1fr 300px 1fr;
      grid-template-rows: 100px 200px;
      grid-gap: 20px;
      grid-auto-flow: column;
   }

    /* The z-index CSS property specifies the z-order of a positioned element and its descendants.
       When elements overlap, z-order determines which one covers the other.
       An element with a larger z-index generally covers an element with a lower one */

      display: grid;

The Holy Grail Layout with CSS Grid

Grids are fundamental to the design of layouts. Over the Web’s relatively short life-span, developers have been trying all kinds of methods and hacks for laying out content in the browser. We started off with no way to do page layout, to table-based layouts, followed by float-based layouts. But those approaches were, at best, workarounds, because neither tables nor floats were meant to be used as true layout tools.

CSS

  
  /* Defining the grid with the grid-area property */
  .grid-header { grid-area: header; }
  .grid-main { grid-area: main; }
  .grid-left { grid-area: menu; }
  .grid-right { grid-area: side; }
  .grid-footer { grid-area: footer; }
  
  /* Specifying the layout of the grid in a visual way using 
     grid-template-areas property. On smaller devices, 
     all the grid items should be displayed in a single column. */

  .container-1 {
    display: grid;
    grid-template-areas: 
      "header"
      "menu"
      "main"
      "side"
      "footer";
    grid-template-columns: 100%;
    grid-template-rows: 100px 50px 1fr 50px 40px;
    min-height: 50vh;
  }
  
  /* Making it Responsive :  On bigger devices, all the grid items 
     should be placed in 3 columns layout */

  @media screen and (min-width: 500px) {
    .container-1 {
      grid-template-areas: 
        "header header header"
        "menu   main   side"
        "footer footer footer";
      grid-template-columns: 15% 1fr 15%;
      grid-template-rows: 100px 1fr 50px;
      grid-gap: 3px;
    }  
  }

  
            
  
    
  /* Defining three columns : the first and last are gutters 
     with a minimum size of 3em but the ability to stretch to fill 1fr.
     Between them is the main content column which can stretch to 
     a maximum width of 40em.
     
     Assigning names in brackets which are entirly optional, 
     but they would make the rest of our styles easier to maintain. */

  .container-2 {
    display: grid;
    grid-template-columns: 
      [full-start] minmax(3em, 1fr) 
      [main-start] minmax(0, 40em) [main-end] 
      minmax(3em, 1fr) [full-end];
  }
  
  /* Instructing all child elements of container-2 to occupy the main column */

  .container-2 > * {
    grid-column: main;
  } 

  /* Making a class for specific elements we'd like to occupy the full container width */

  .container-2 > .full-image {
    grid-column: full;
  }
