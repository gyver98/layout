Grid

Two-dimensional layout system

CSS Grid provides a trivial solution to a complex problem: How to display full-bleed content mixed with center-aligned width-constrained content. This solution elimiates the need for auto-margins and floats while allowing the developer to change the grid globally and see those changes apply to each element individually.

  • Grid Lines

    Grid lines are the horizontal and vertical dividing lines of the grid. A grid line exists on either side of a column or row. They can be referred to by numerical index, which starts at 1.

  • Grid Tracks

    Grid track is a generic term for a grid column or grid rowâ€”in other words, it is the space between two adjacent grid lines.

  • Grid Areas

    Grid area is the logical space used to lay out one or more grid items. A grid area consists of one or more adjecent grid cells.

Grid Container: Defining the Grid Structure

The structure of the grid, like how many rows and columns it has and their sizing, is controlled by properties applied to the grid container. Placement of grid items is determined by CSS properties applied to the child elements inside the grid container.

A grid container is defined by setting the display to grid or inline-grid on an element. This creates a grid formatting context for its content, which are laid out into a grid. The grid formatting context only applies to child elements and does not extend to grandchild elements and beyond.

Explicit Grid

The explicit grid is what we define using the relevant CSS grid properties, like grid-template-rows, grid-template-columns and grid-template-areas.

Implicit Grid

When grid items are positioned outside of these bounds, the grid container generates implicit grid tracks by adding implicit grid lines to the grid.
